Pasta para a Web APi

Correr mongoDB no docker: 
v1:
  docker run -d --name mongodb  -p 27017:27017 mongo
  docker ps
  docker exec -it <id_container>  mongosh
    show dbs
    use moviesdb
    show collections
v2: GUI
  docker network create movies-net
  docker run -d --name mongodb --network movies-net -p 27017:27017  -v mongo_data:/data/db mongo
  docker run -d --name mongo-express --network movies-net -p 8081:8081  -e ME_CONFIG_MONGODB_SERVER=mongodb -e ME_CONFIG_MONGODB_PORT=27017 -e ME_CONFIG_BASICAUTH=false  mongo-express
  docker ps *deve estar mongodb e mongo-express
  ir a http://localhost:8081


usar MongoDB for VS Code

Endpoints:
Movies:
GET    /movies
POST   /movies
PUT    /movies?:id
DELETE /movies?:id

User:
GET    /users
POST   /users
POST /users/login - Login

History:
GET /users/:id/history
POST /history/watch
PUT /history/ID - mudar data de watched
DELETE /history/ID

GET /history/unwatched/:id



para correr containers: docker compose up --build
para parar: docker compose down

Docker Hub - guarda as imagens backend e frontend
no terminal (depois de criar conta docker):
	- docker login
	para RestApi
	- docker build -t movies-backend .
	para WebApp
	- docker build -t movies-frontend .
	criar tags:
	docker tag movies-backend «username»/movies-backend:latest
	docker tag movies-frontend «username»/movies-frontend:latest

	docker push

  	adicionar image path to docker compose
